# Environment Variables Configuration

## Quick Setup

Copy `.env.example` to `.env`:

```bash
cp .env.example .env
```

## Variables

### Gateway Configuration

| Variable | Default | Purpose |
|----------|---------|---------|
| `OPENCLAW_GATEWAY_TOKEN` | `dev-openclaw-token-18789` | Authentication token cho gateway (local dev) |
| `OPENCLAW_GATEWAY_URL` | `ws://localhost:18789` | Gateway WebSocket URL |
| `OPENCLAW_GATEWAY_BIND` | `loopback` | Gateway bind address (loopback/lan) |
| `OPENCLAW_GATEWAY_PORT` | `18789` | Gateway port |

### Server Configuration

| Variable | Default | Purpose |
|----------|---------|---------|
| `PORT` | `18788` | Onboard server port |
| `GATEWAY_PORT` | `18789` | Gateway port (used to start gateway) |
| `NODE_ENV` | `development` | Node environment |

### Optional: API Keys for Onboarding

Nếu muốn pre-fill API key trong onboard process (không recommended cho prod):

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_GEMINI_API_KEY=AIza...
OPENAI_API_KEY=sk-...
```

⚠️ **Security Note:** Không commit `.env` vào git (nó đã trong `.gitignore`).

## Development Workflow

```bash
# 1. Setup
cp .env.example .env

# 2. Chạy onboard server (tự động load .env)
npm run onboard

# 3. Browser sẽ dùng token từ .env khi cần
```

---

## Token Formats

### Local Development

```bash
OPENCLAW_GATEWAY_TOKEN=dev-openclaw-token-18789
```

### Production (từ Railway/hosting)

```bash
OPENCLAW_GATEWAY_TOKEN=prod-token-xxx-yyy-zzz
```

Token được generate bởi OpenClaw khi setup gateway.

## Troubleshooting

### Gateway token mismatch
- Kiểm tra: `~/.openclaw/config.json` (chứa actual token)
- Update `.env` nếu token thay đổi

### Connection refused
- Kiểm tra port: `lsof -i :18789` (Mac/Linux) hoặc `netstat -an | find "18789"` (Windows)
- Thay đổi `GATEWAY_PORT` trong `.env` nếu port bị dùng

### .env không được load
- Xóa cache: `rm -rf ~/.openclaw/` 
- Restart onboard server: `npm run onboard`
